[ HOSTNAME: Raiden ]

MSF Compute Server


Raiden is the primary compute server in MSF, it is a refurbished Dell Precision 5810. I use Proxmox VE 8 for the hypervisor (soon EOL, so I will be upgrading to PVE 9), the specific hardware is listed below:

[CPU]: Intel E5-1650 | 6C/12T
[GPU]: NVIDIA Quadro P4000
[RAM]: 64 GB DDR4
[STORAGE]: 2x1TB HDD ZFS, 1x500GB HDD (Boot/System)
[MODEL]: DELL Precision 5810
[OS]: Proxmox VE 8+
[LXC/VM OS]: Debian 12

This currently runs all of the LXCs and VMs that make up MSF. This makes up all of my Docker containers, services, as well as other core networking Infrastructure like Nginx Proxy Manager and AdGuardHome. Eventually I would like to setup a HA cluster using 3 identical Thin Clients, or NUCs. This would split up the workload among 3 compute nodes, as well as offer resilience and failover for the network.

Raiden originally started off as a second-hand thinkstation, though the compute power was lacking leading me to upgrade it to the 5810 I have today.

[ HOSTNAME: Motherbase ]

MSF NAS Storage


Motherbase is the primary storage server in MSF, it is a third-hand Dell PowerEdge R510. I use TrueNAS Scale for the OS, the specific hardware is listed below:

[CPU]: 2xIntel E5640 | 8C/16T Total
[RAM]: 128GB DDR3 ECC
[STORAGE]: 12x4TB HDD (ZFS Pools), 2x500GB HDD (Cache), 2x64GB Cruzer Glide (Boot)
[HBA]: LSI SAS2008 (IT Mode)
[BACKPLANE]: SAS2
[NIC]: 2xIntegrated NIC in LAG
[MODEL]: DELL PowerEdge R510
[OS]: TrueNAS Scale CE 24+

[POOL_0]: 1 x RAIDZ1, 3 wide, 4 TB | 1x 500GB Cache
[POOL_1]: 1 x RAIDZ1, 3 wide, 4 TB
[POOL_2]: 1 x RAIDZ1, 6 wide, 4 TB | 1x 500GB Cache

This currently is being used as just a storage server. While it has more theoritcal resources than Raiden, I don't use it for any compute as 48TB of ZFS Storage makes good use of the RAM installed, I don't want to pull that capacity for docker containers or other compute - though I have been looking into MINIO under TrueNAS. It using DDR3 means that the 128GB of RAM is not as impressive as the 64GB of DDR4 on Raiden, so that itself makes it less compatible for compute tasks. Eventually I would like to scale this up as well, but the R510 has served me well for storage, the only think I could really care to benefit from would be increased storage size. I intend to continue a non-converged setup, so adding compute power is not a priority for me.

[ HOSTNAME: Otacon ]

Mobile Dev, Programming Thinkpad


After owning many different thinkpads (and many different devices named Otacon) over the years, I've settled on the X220. It is the last Thinkpad with the legacy "tactile" keyboard, it is super comfortable to type on, very enjoyable "feel" and I'm not deprived of many of the keys the later keyboards would deprecate. In addition to this, while it is a 15 year old laptop, after a repaste and a subtle RAM upgrade performs great for "on-the-go" work. It has been my go-to mobile workstation for many years now.

>> ARCHIVED | My first Thinkpad


This was my first ever Thinkpad, due to it's age and lack of compute-power (e.g. 2GB DDR2-SODIM) it was very difficult to use for more than web-browsing or terminal sessions, though this did lead to me leaning into very low-resource terminal alternatives to programs. Some of these follow me to this day: i3, urxvt, and polybar. This initiated my enjoyment of ricing and maximizing resource-effeciency where I could. This was not my first time using Linux, thankfully, but it was my first time using a TWM and really leaning into the customization available to me. Before the R61i, I mostly just used stock KDE Plasma 5.
