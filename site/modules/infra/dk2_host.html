Prometheus, NodeExporter, Cadvisor stack with network segmentation.

In order to get actual statistics to my website, instead of a bunch of placeholder information; I am currently working on a Prometheus stack. The plan is to have Prometheus run on my MGT node, while NodeExporter and Cadvisor will run on DK3 - which are in seperate VLANs. The main reason for this is "Out-of-Band" monitoring. So that if my Docker Swarm ever crashes due to a resource leak, the Prometheus instance will remain alive to tell me exactly what happened right before.

The plan is to scrape all the data from Proxmox, TrueNAS Scale, and my Portainer Swarm so that I may use the data on the site, without having to write dozens of API calls through Nginx. This will be faster, safer, and give me a chance to run a production monitoring setup.

I will keep this updated with any progress made as well as any disasters in the process.
