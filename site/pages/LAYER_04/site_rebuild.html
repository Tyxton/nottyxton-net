>> JAVASCRIPT_CHALLENGE

When building the boot animation, I ran into an issue where everytime the page loaded, it would replay, even on page redirect. This was an issue I wanted to fix immediately since, while unique and visually appealing, looses its charm quickly when viewed dozens of times per visit. Especially when most visitors frequently jump between pages.


Any recruiters visiting my page specifically would be deterred by this, their time is valuable, so I had to figure something out.


>> THE_SOLUTION

I implemented a session state check in the script, unlike localStorage, sessionStorage clears itself when the tab is closed meaning that the boot sequence will still play for the first time a user visits the website, but won't annoy them as they click through the various pages. This too, would allow for returning users to still get the animation, as it is a charm of the site.


To compliment the session state check, I also added a skipping feature so that when a user clicks, it will automatically skip the boot animation, and just show the content -- that way users don't feel forced to watch it. User satisfaction is a crucial part in UI/UX development afterall.


>> KEY_LEARNINGS

In writing this script, I learned a key skill in web development -- Session State Management to compliment my skills in UI/UX design.
